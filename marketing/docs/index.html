Quick Start

Get up and running with G-Rump in under 5 minutes.

# Clone the repository
git clone https://github.com/your-org/g-rump.git
cd g-rump

# Install dependencies
npm install

# Start the development server
npm run dev

Installation

Intent Parsing

The intent compiler is the heart of G-Rump. It uses a Rust-based WASM parser to extract structured requirements from natural language.

Example Intent

"Build a subscription analytics dashboard for SaaS 
companies with Stripe integration, user authentication,
and real-time charts using React and Node.js"

Extracted Fields

  • Features: analytics dashboard, user authentication, real-time charts
  • Integrations: Stripe
  • Tech Stack: React, Node.js
  • Users: SaaS companies

Architecture Generation

Once intents are parsed, G-Rump generates a complete system architecture including:

  • C4 Model diagrams (Context, Container, Component)
  • Database schema recommendations
  • API endpoint specifications
  • Technology stack validation

Code Generation

G-Rump generates production-ready code with:

  • TypeScript type definitions
  • Error handling and validation
  • Test scaffolding
  • Documentation comments

API Overview

The G-Rump API provides programmatic access to all features. Base URL:

https://api.g-rump.com/v1

Endpoints

POST /architecture/generate

Generate system architecture from a product description.

curl -X POST https://api.g-rump.com/v1/architecture/generate \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "projectDescription": "Build a task management app",
    "projectType": "web"
  }'

POST /code/generate

Generate code from architecture or PRD.

Authentication

All API requests require authentication via Bearer token:

Authorization: Bearer YOUR_API_KEY

Get your API key from the dashboard.
